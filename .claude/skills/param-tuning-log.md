# Nav2 파라미터 튜닝 기록 스킬 (param-tuning-log)

> **트리거**: 아래 표현이 나오면 이 스킬을 즉시 실행한다.
> - "파라미터 튜닝" / "nav2 조정" / "파라미터 변경"
> - "costmap 조정" / "MPPI 조정" / "velocity 리밋"
> - nav2_params.yaml 파라미터 변경 작업 착수 시

---

## 목적

Nav2 파라미터(costmap 해상도, MPPI 가중치, velocity_smoother 리밋 등)를 변경할 때 before/after를 체계적으로 기록하여 "왜 이 값인가"를 추적 가능하게 한다.

---

## 대상 파일

| 파일 | 주요 파라미터 |
|------|-------------|
| `src/ad_bringup/config/nav2_params.yaml` | Nav2 전체 파라미터 |
| `src/ad_bringup/config/dual_ekf.yaml` | EKF 필터 파라미터 |
| `docs/arch/ARCH-003_nav2_stack.md` | Nav2 아키텍처 결정 기록 |

---

## 실행 순서 (순서 준수 필수)

### STEP 1 — 변경 전 스냅샷

변경 대상 파라미터의 현재 값을 기록한다:

```
📸 파라미터 스냅샷 (BEFORE)
파일: src/ad_bringup/config/nav2_params.yaml
섹션: velocity_smoother

| 파라미터 | 현재값 | 단위 |
|---------|--------|------|
| max_velocity[0] | 0.5 | m/s |
| max_velocity[1] | 0.0 | m/s |
| max_velocity[2] | 1.0 | rad/s |
| max_accel[0] | 0.3 | m/s² |
```

파일에서 해당 섹션을 읽어 테이블로 정리.

### STEP 2 — 변경 사유 기록

반드시 다음 질문에 답한다:

1. **어떤 문제를 해결하려는가?** (현상 설명)
2. **왜 이 파라미터를 바꾸려는가?** (인과 관계)
3. **기대 효과는?** (정량적 목표)

예시:
```
사유: 직선 구간에서 최대 속도 0.5m/s가 너무 느림.
      목표 경로 완주 시간 60s → 40s로 단축.
      max_velocity[0]을 0.8m/s로 증가 시도.
```

### STEP 3 — 파라미터 변경

사용자와 합의된 값으로 YAML 파일을 수정한다.

### STEP 4 — 변경 후 스냅샷

```
📸 파라미터 스냅샷 (AFTER)
파일: src/ad_bringup/config/nav2_params.yaml
섹션: velocity_smoother

| 파라미터 | 이전값 | 변경값 | 단위 | 변화 |
|---------|--------|--------|------|------|
| max_velocity[0] | 0.5 | 0.8 | m/s | +60% |
| max_accel[0] | 0.3 | 0.5 | m/s² | +67% |
```

### STEP 5 — 테스트 결과 기록 (정량 메트릭 필수)

파라미터 변경 후 테스트를 수행하고 결과를 기록한다:

```
📊 테스트 결과

| 메트릭 | 변경 전 | 변경 후 | 목표 | 판정 |
|--------|--------|--------|------|------|
| 경로 완주 시간 | 60s | 42s | <45s | ✅ |
| 최대 횡가속도 | 0.2m/s² | 0.35m/s² | <0.5 | ✅ |
| 경로 이탈 횟수 | 0 | 0 | 0 | ✅ |

결론: max_velocity 0.8m/s 적용 가능. 안정성 유지됨.
```

테스트를 바로 수행할 수 없는 경우 → "테스트 미수행, 다음 시뮬레이션에서 검증 예정"으로 기록.

### STEP 6 — 문서 기록

변경 규모에 따라 기록 위치를 결정한다:

| 규모 | 기록 위치 |
|------|----------|
| 단순 값 조정 (1~2개 파라미터) | TASK.md 현재 작업에 한 줄 기록 |
| 다수 파라미터 연동 변경 | `docs/프로젝트/task/{TASK_ID}.md` 상세 로그 |
| Nav2 구조 변경 (플러그인 교체 등) | `docs/arch/ARCH-003_nav2_stack.md` ADR 업데이트 |

### STEP 7 — ADR 업데이트 판단

`arch-management.md` A1~A5 기준으로 ARCH-003 업데이트 필요 여부를 판단:

- 플러그인 교체 (DWA→MPPI 등) → ADR 업데이트 **필수**
- costmap 해상도 변경 → 성능-안전 트레이드오프(A4) → ADR 업데이트 **권장**
- velocity 리밋 미세 조정 → ADR 불필요, TASK.md로 충분

## 주의사항

- 파라미터 변경은 **빌드 없이** 반영됨 (YAML 파일 → launch 시 로드). 단, 이미 실행 중인 노드에는 영향 없음
- 여러 파라미터를 동시에 바꾸면 인과 관계 추적이 어려움 → 가능하면 **한 번에 하나씩** 변경
- `velocity_smoother` 관련 변경은 C61 블로커와 연관될 수 있음 → `dds-troubleshoot` 스킬 참조
- 실차 적용 전 시뮬레이션에서 반드시 검증
