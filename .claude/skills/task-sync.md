# TASK 이슈 동기화 스킬 (task-sync)

> **트리거**: 아래 표현이 나오면 이 스킬을 즉시 실행한다.
> - "TASK 동기화" / "이슈 동기화" / "업무 동기화"
> - "노션 이슈 싱크" / "개인 노션 동기화"
> - standup 브리핑 후 자동 실행 가능

---

## 목적

PLUVA Robotics 팀 **TASK 이슈 관리 DB**에서 전력제어개발팀 담당 이슈를 가져와
개인 **통합 업무 관리 DB**에 동기화한다.

## 실행 순서 (순서 준수 필수)

### STEP 1 — 팀 TASK 이슈 검색

TASK 이슈 관리 DB에서 **전력제어** 관련 이슈를 검색한다.

```
notion-search:
  query: "전력제어"
  data_source_url: "collection://2dc3eb8a-f0d1-81be-af25-000b6e466bcb"
```

검색 결과에서 각 이슈 페이지를 `notion-fetch`로 조회하여 다음을 수집:
- ISSUE # (자동 번호)
- 항목명 (1.2.5 = TITLE)
- 담당부서 (multi_select) — "전력제어" 포함 여부 확인
- 상태 (multi_select)
- 중요도
- (목표)완료일
- 주요내용
- 카테고리

### STEP 2 — 개인 DB 현황 조회

통합 업무 관리 DB에서 PLUVA Robotics 항목을 검색한다.

```
notion-search:
  query: "PLUVA Robotics"
  data_source_url: "collection://45c7bc27-788f-439a-bf9d-0f5daaa98de2"
```

`[#XX]` 패턴으로 시작하는 항목을 추출하여 ISSUE # 매핑 테이블 생성.

### STEP 3 — 비교 & 동기화

각 팀 이슈에 대해:

| 조건 | 행동 |
|------|------|
| 개인 DB에 없는 새 이슈 | `notion-create-pages`로 추가 |
| 상태가 변경된 이슈 | `notion-update-page`로 상태 갱신 |
| 마감일이 변경된 이슈 | `notion-update-page`로 마감일 갱신 |
| 개인 DB에만 있고 팀에 없는 이슈 | 메모에 "[팀 DB에서 삭제됨]" 추가, 확인 요청 |
| 변경 없음 | 스킵 |

### STEP 4 — 변경사항 브리핑

다음 형식으로 사용자에게 보고한다:

---

**🔄 TASK 이슈 동기화 — {날짜}**

**➕ 신규 등록**
- `[#XX]` {항목명} — {상태}, {우선순위}

**🔃 상태 변경**
- `[#XX]` {항목명}: {이전 상태} → {현재 상태}

**📅 마감일 변경**
- `[#XX]` {항목명}: {이전 마감일} → {현재 마감일}

**✅ 변경 없음**: {N}건

**📊 전력제어 이슈 현황**: 진행 {N} / 예정 {N} / 보류 {N} / 완료 {N}

## 매핑 규칙

### 상태 매핑 (팀 DB → 개인 DB)

| 팀 DB 상태 | 개인 DB 상태 | 비고 |
|------------|-------------|------|
| 진행 중 | 진행중 | |
| 시작 전 | 예정 | |
| 보류 | 보류 | |
| 완료 | 완료 | |
| 지연 | 진행중 | 메모에 `[지연]` 표기 |
| DROP | 보류 | 메모에 `[DROP]` 표기 |

### 속성 매핑 (팀 DB → 개인 DB)

| 팀 DB 속성 | 개인 DB 속성 | 변환 |
|------------|-------------|------|
| 1.2.5 (TITLE) | 항목명 | `[#XX] {제목}` 형식 |
| 상태 | 상태 | 위 매핑 테이블 참조 |
| (목표)완료일 | 마감일 | 그대로 복사 |
| 주요내용 | 메모 | 그대로 복사 |
| 중요도 | 우선순위 | 높음→P1, 보통→P2, 낮음→P3 |
| 원본 페이지 URL | Notion 링크 | `https://www.notion.so/{page_id}` |
| — | 프로젝트 | 항상 `PLUVA Robotics` |
| — | 항목 유형 | 기본 `업무` (카테고리가 "시험/검증"이면 `테스트`) |

### 우선순위 매핑 (팀 DB 중요도 → 개인 DB)

| 팀 DB 중요도 | 개인 DB 우선순위 |
|-------------|-----------------|
| 긴급 / 높음 / High | P1 긴급 |
| 보통 / Medium | P2 중요 |
| 낮음 / Low | P3 일반 |
| (비어있음) | (비어있음) |

## 주의사항

- **회의 항목은 동기화 대상이 아님**: 전력제어개발팀 주간회의, PLUVA Robotics 주간회의 등 `항목 유형=회의`인 항목은 수동 관리
- **회사 잡무 항목은 동기화 대상이 아님**: 프로젝트=회사 잡무인 항목은 별도 관리
- **담당부서 필터**: 반드시 `전력제어`가 담당부서에 포함된 이슈만 동기화
- **중복 방지**: ISSUE #로 매칭. `[#XX]` 패턴이 항목명에 없으면 신규로 판단
- **삭제 금지**: 개인 DB 항목을 자동 삭제하지 않음. 팀 DB에서 사라진 경우 메모로 알림
