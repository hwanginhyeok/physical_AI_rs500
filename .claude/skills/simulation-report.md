# 시뮬레이션 테스트 리포트 작성 스킬

> 트리거: 모듈 시뮬레이션 결과를 리포트로 정리할 때

## 네이밍 규칙

```
results/reports/YYMMDD_블록명_report.md
results/reports/figures/YYMMDD_블록명_*.png
```

- `YYMMDD`: 작성일 (예: 260224)
- `블록명`: 모듈의 영문 식별자 (drivetrain, terrain, dynamics, sensor)
- 예시: `260224_drivetrain_report.md`

## 리포트 구조

```markdown
# [모듈명] 시뮬레이션 테스트 리포트

> 작성일: YYYY-MM-DD
> 모듈: `소스파일 경로`

## 1. 모듈 개요

- **목적**: 한 줄 설명
- **I/O 정의**:
  | 구분 | 항목 | 타입 | 단위 |
  |------|------|------|------|
  | Input | ... | ... | ... |
  | Output | ... | ... | ... |

## 2. 평가 지표

| 지표 | 정의 | 수식 | 목표 |
|------|------|------|------|
| 지표1 | ... | ... | ... |

## 3. 테스트 시나리오

각 시나리오별:
- 입력 조건
- 기대 동작 (해석해 또는 물리적 기대값)

## 4. 파라미터 감도 분석

- **Sweep 대상**: 파라미터명 (범위, N단계)
- **감도 히트맵/곡선**: 그림 참조
- **최적점**: 지표 기준 최적 파라미터 조합

## 5. 결과 요약

| 시나리오 | 지표1 | 지표2 | 지표3 | 판정 |
|----------|-------|-------|-------|------|

## 6. 결론 및 한계

- 주요 발견
- 알려진 한계
- 향후 개선 방향
```

## 작성 원칙

1. **정량적 기술**: 모든 결과는 수치로 표현. "좋다/나쁘다" 대신 RMSE=0.023m
2. **재현 가능**: 실행 명령어, 시드, 파라미터를 명시하여 누구나 재현 가능
3. **그림 필수**: 감도 분석은 반드시 시각화 (히트맵, 곡선) 첨부
4. **기준 명시**: 평가 지표의 "좋음/나쁨" 판단 기준을 사전 정의
5. **단위 통일**: SI 단위 사용 (m, s, N, rad, kg)

## 평가 지표 선정 기준

모듈당 2~3개, 다음 관점에서 선정:

- **정확성**: 해석해 또는 이론값 대비 오차
- **안정성**: 발산/진동 없이 수렴하는가
- **물리적 타당성**: 에너지 보존, 힘 균형 등 물리 법칙 위배 여부

## 파라미터 감도 분석 절차

1. Sweep 대상 파라미터 2~3개 선정
2. 각 파라미터를 N단계 (기본 10)로 등간격 분할
3. 모든 조합에서 시뮬 실행 (2개면 N^2, 3개면 N^3)
4. 지표별 감도 매트릭스 생성
5. 최적 조합 = 모든 지표가 목표 범위 내인 파라미터 세트
