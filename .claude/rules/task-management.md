# TASK 관리 — 그린 책임

> 태스크는 **3파일 체제**로 관리한다.
> "나중에 정리"가 아니라 **작업 흐름 속에서 실시간으로** 갱신한다.

## 파일 구조

| 파일 | 용도 | 테이블 컬럼 |
|------|------|-------------|
| `CURRENT_TASK.md` | 진행 중 | `# / 태스크 / 시작일 / blocked / 비고` |
| `PREPARED_TASK.md` | 예정 + TODO | `# / 태스크 / 우선순위 / depends / 비고` |
| `FINISHED_TASK.md` | 당월 완료 | `# / 태스크 / 완료일 / 비고` |
| `TASK.md` | 인덱스 (요약) | 링크 + 개수 |
| `TASK_ARCHIVE/YYYY-MM.md` | 월별 아카이브 | 과거 완료 항목 |
| `DIFFICULTY.md` | 어려웠던 문제 & 노하우 | 삽질 기록, 해결책 |

- **우선순위**: P1(긴급) > P2(중요) > P3(향후)
- **상태 흐름**: PREPARED → CURRENT → FINISHED
- **TODO**: PREPARED_TASK.md 하단에서 관리

## 그린의 역할

- 회의/대화에서 나온 **모든 task를 즉시** PREPARED_TASK.md에 등록
- 세션 시작 시 CURRENT_TASK.md 점검, blocked 항목 보고
- 새로운 요청·버그·개선 포인트 발견 시 대화 중이라도 즉시 등록
- 삽질/어려운 문제 해결 시 DIFFICULTY.md에 기록

## 실시간 갱신 규칙 (필수)

| 트리거 | 행동 |
|--------|------|
| **작업 착수 시** | PREPARED → CURRENT로 이동 + 시작일 기입 |
| **작업 완료 시** | CURRENT → FINISHED로 이동 + 완료일 기입 + TASK.md 요약 갱신 |
| **새 작업 발견 시** | PREPARED_TASK.md에 즉시 등록 |
| **커밋 직전** | TASK 상태가 실제와 일치하는지 점검 |
| **세션 종료 시** | CURRENT에 blocked/진행 상황 업데이트 |
| **월초** | 전월 FINISHED → TASK_ARCHIVE/YYYY-MM.md로 이동 |
| **어려운 문제 해결 시** | DIFFICULTY.md에 D-NNN 항목 추가 |

> 완료된 작업이 CURRENT에 남아있으면 안 되고, 진행 중인 작업이 기록 없이 진행되면 안 된다.

## TASK 상세 로그 — `docs/프로젝트/task/{TASK_ID}.md`

모든 TASK에 쓸 필요 없음. **아래 경우에만** 작성:

- 설계 결정이 포함된 작업 (왜 A를 선택하고 B를 기각했는지)
- 여러 파일을 변경한 작업 (변경 추적 필요)
- 나중에 "왜 이렇게 했지?" 질문이 예상되는 작업
- 사용자 검토가 필요한 작업 (미결 이슈 기록)

**작성 시점**: 작업 완료 시 또는 중요 결정이 내려진 시점

**템플릿**: `docs/프로젝트/task/TEMPLATE.md` 참조

**필수 섹션**: 배경, 결정 사항(결정+이유+대안), 변경 내역, 검증, 미결 이슈

## DIFFICULTY.md 기록 규칙

**아래 경우에 기록**:
- 30분 이상 삽질한 문제
- 같은 실수를 반복할 가능성이 있는 문제
- 해결 방법이 직관적이지 않은 문제

**포맷**: 날짜, 상황, 이슈, 삽질, 해결, 대안, 노하우, 회고, 관련 파일
