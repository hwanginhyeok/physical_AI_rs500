# Application 정의서: RS500 궤도차량 자율주행 시스템

> 작성일: 2026-02-21

---

## 1. 시스템 개요

**RS500 궤도차량 기반 모듈식 자율주행 플랫폼**

RS500 궤도차량에 자율주행 소프트웨어를 탑재하여, 비포장/험지 환경에서
자율적으로 임무를 수행하는 시스템. 하드웨어 플랫폼은 고정하되,
소프트웨어는 모듈식으로 설계하여 적용 분야 확장이 용이한 구조.

---

## 2. 적용 분야

### Phase 1 (현재 목표): 농업용 자율주행
- 농경지 자율 주행 (밭, 과수원, 비닐하우스 간 이동)
- 작물 관리 경로 자동 생성
- 방제/파종 장비 연동
- GPS 기반 구역 관리

### Phase 2 (확장 목표): 군사/탐사용
- 비포장/험지 정찰
- 물자 수송 (무인 보급)
- 지형 탐사 및 매핑
- GPS-denied 환경 대응

---

## 3. 하드웨어 플랫폼 (고정)

| 구성 | 사양 |
|------|------|
| 차체 | RS500 궤도차량 (1톤급) |
| 크기 | 2.0m(L) x 1.2m(W) x 0.8m(H) |
| 구동 | 좌/우 트랙 독립 구동 (스키드 스티어링) |
| 센서 | GPS, IMU, 전방 카메라, LiDAR |
| 통신 | ROS2 기반 |

---

## 4. 소프트웨어 아키텍처 (모듈식)

```
┌─────────────────────────────────────────────┐
│              Application Layer               │
│  (농업 미션 / 군사 미션 / 탐사 미션 ...)      │
├─────────────────────────────────────────────┤
│              Planning Layer                  │
│  경로 계획 │ 임무 관리 │ 행동 결정            │
├─────────────────────────────────────────────┤
│             Perception Layer                 │
│  장애물 감지 │ 지형 인식 │ 위치 추정(SLAM)    │
├─────────────────────────────────────────────┤
│              Control Layer                   │
│  속도 제어(PID) │ 조향 제어 │ 트랙 구동       │
├─────────────────────────────────────────────┤
│            Hardware Abstraction              │
│  센서 드라이버 │ 액추에이터 │ ROS2 인터페이스  │
└─────────────────────────────────────────────┘
```

### 핵심 설계 원칙
- **모듈 독립성**: 각 레이어는 ROS2 토픽/서비스로만 통신, 내부 구현 교체 가능
- **미션 플러그인**: Application Layer를 교체하면 다른 분야에 즉시 적용
- **시뮬레이션 우선**: Gazebo에서 검증 후 실차 배포 (SIL → HIL → 실차)

## 5. 핵심 모듈 정의

### 5.1 인지 (Perception)
| 모듈 | 입력 | 출력 | 비고 |
|------|------|------|------|
| 장애물 감지 | LiDAR, Camera | 장애물 목록 (위치, 크기) | 포인트클라우드 + 이미지 융합 |
| 지형 분류 | Camera, LiDAR | 지형 타입 (포장/비포장/경사) | 주행 가능 영역 판단 |
| 위치 추정 | GPS, IMU, LiDAR | 차량 위치/자세 | GPS 가용 시 RTK, 불가 시 SLAM |
| 차선/경계 인식 | Camera | 경로 경계선 | 농경지 고랑/경계 인식 |

### 5.2 판단 (Planning)
| 모듈 | 입력 | 출력 | 비고 |
|------|------|------|------|
| 전역 경로 계획 | 목적지, 지도 | 웨이포인트 경로 | A*, RRT* 등 |
| 지역 경로 계획 | 장애물, 현재 위치 | 회피 경로 | DWA, TEB 등 |
| 행동 결정 | 인지 결과 전체 | 주행 모드 | FSM 또는 BT |
| 임무 관리 | 사용자 명령 | 세부 작업 시퀀스 | 미션별 플러그인 |

### 5.3 제어 (Control)
| 모듈 | 입력 | 출력 | 비고 |
|------|------|------|------|
| 속도 제어 | 목표 속도 | 트랙 속도 명령 | PID 제어 |
| 경로 추종 | 목표 경로 | 조향 명령 | Pure Pursuit / Stanley |
| 트랙 구동 | 속도+조향 | 좌/우 트랙 RPM | 스키드 스티어 변환 |
| 비상 정지 | 장애물 근접 | 즉시 정지 | 최우선 안전 모듈 |

## 6. 개발 및 검증 프로세스

```
Phase 1: 시뮬레이션 (현재)
  Gazebo gz-sim → ROS2 → 알고리즘 개발/검증

Phase 2: SIL (Software-in-the-Loop)
  실차 모델 정밀화 → 시나리오 기반 테스트

Phase 3: HIL (Hardware-in-the-Loop)
  실차 ECU + 시뮬레이션 센서 → 제어기 검증

Phase 4: 실차 테스트
  실제 농경지/험지 → 현장 검증
```

## 7. 확장성 설계

| 변경 항목 | 영향 범위 | 방법 |
|-----------|-----------|------|
| 새 센서 추가 | HAL + Perception | ROS2 드라이버 추가, 브릿지 설정 |
| 미션 변경 (농업→군사) | Application Layer만 | 미션 플러그인 교체 |
| 경로 알고리즘 변경 | Planning Layer만 | 동일 인터페이스로 교체 |
| 제어 튜닝 | Control Layer만 | 파라미터 파일 수정 |
| 차량 변경 | HAL + SDF 모델 | 새 URDF/SDF + 파라미터 |
