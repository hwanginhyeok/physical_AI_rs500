# C50: MDROBOT MD2K 모터 컨트롤러 반영 + C49 정정 + P1 후속작업

- 분야: 시뮬레이션
- 담당: 그린
- 기간: 2026-03-25
- 상태: 완료

## 배경

C49 리포트에서 모터 컨트롤러를 Curtis 1226BL로 기술했으나, 실차는 **MDROBOT MD2K** (듀얼채널 DC 모터 드라이버)를 사용함을 확인. Curtis 관련 내용을 전부 MD2K로 정정하고, MD2K의 제어 특성을 시뮬레이션에 반영함.

동시에 C49에서 도출된 P1 후속작업(비상정지, 자율주행 모드, 시스템 폴트, 브레이크)을 구현.

## 결정 사항

| 결정 | 이유 | 검토한 대안 |
|------|------|-------------|
| `autonomous_mode` 디폴트 `True` | 기존 218건 테스트 깨짐 방지. 시뮬레이션 환경에서는 자율모드가 기본. 실차 연동 시 명시적으로 `set_autonomous_mode(False)`로 시작 | 디폴트 `False`: 안전 우선이지만 기존 코드 전부 수정 필요 |
| SS/SD=0이면 기존 1차 지연 유지 | 하위호환. 기존 동작을 바꾸지 않으면서 MD2K 램프 기능을 opt-in으로 제공 | SS/SD를 항상 적용: 기존 동작이 바뀌어 218건 테스트 영향 |
| 안전 상태를 VehicleDynamics에 구현 | 차량 레벨에서 명령을 차단하는 것이 가장 상위 레이어. DrivetrainModel은 모터 물리만 담당 | DrivetrainModel에 구현: 모터 레벨 차단이지만, 차량 관성에 의한 감속을 반영 못함 |
| 보호기능을 별도 모듈(motor_protection.py)로 분리 | SingleMotorModel의 책임이 과도해짐 방지. 보호 기능은 선택적 주입으로 설계 | SingleMotorModel 내부에 통합: 클래스가 비대해지고 테스트 어려움 |

## 파라미터 출처 및 가정

> **이 섹션은 C50에서 설정한 모든 파라미터의 출처, 역산 근거, 가정을 기록한다.**
> 실차 데이터 확보 시 반드시 재검증해야 한다.

### DrivetrainConfig 신규 파라미터

| 파라미터 | 값 | 출처 | 가정/한계 |
|---------|-----|------|---------|
| `rated_voltage` | 48.0 V | DB130-48 데이터시트 "DC 48V" 명기 | 확정값 |
| `rated_current` | 78.0 A | DB130-48 데이터시트 "78A" 명기 | 확정값 |
| `max_current` | 100.0 A | MD2K 매뉴얼 "듀얼채널 100A" 명기 | 확정값. 채널당 100A |
| `slow_start_time` | 0.0 s (디폴트) | 디폴트 0 = 기존 1차 지연 유지 | **가정**: 실차 MD2K SS 가변저항 세팅값 미확인. 실차 확인 필요 |
| `slow_down_time` | 0.0 s (디폴트) | 디폴트 0 = 기존 1차 지연 유지 | **가정**: 실차 MD2K SD 가변저항 세팅값 미확인 |
| `brake_torque` | 16.0 Nm | DB130-48 데이터시트 "DC48V 전자브레이크, 16 Nm" 명기 | 확정값. 모터 축 기준 |
| `over_temp_threshold` | 65.0 °C | MD2K 매뉴얼 "과온도 65°C" 명기 | 확정값. 드라이버 기판 온도 기준 |
| `stall_alarm_threshold` | 0.15 (15%) | MD2K 매뉴얼 "STALL 알람 15%" 명기 | 확정값 |

### ProtectionConfig 열 모델 파라미터

| 파라미터 | 값 | 역산 근거 | 가정/한계 |
|---------|-----|---------|---------|
| `winding_resistance` | 0.05 Ω | **추정**. DB130-48 스펙시트에 권선 저항 미기재. 3kW급 BLDC 모터 일반적 범위 0.02~0.1Ω에서 중간값 사용 | **미확보**. 실측 또는 스펙시트 확인 필요 |
| `thermal_resistance` | 0.002 °C/W | **역산**. 목표: "정격 전류(78A) 연속 운전 시 정상상태 온도 ≈ 60~62°C (65°C 직전)" | 아래 역산 과정 참조 |
| `cooling_time_constant` | 60.0 s | **추정**. 3kW급 금속 하우징 모터의 자연 냉각 시정수 일반적 범위 30~120s에서 중간값 | **미확보**. 실차 온도 로깅으로 튜닝 필요 |
| `soft_fuse_time` | 2.0 s | MD2K 매뉴얼 Soft fuse 관련 기술 기반 | 정확한 초 단위 설정은 MD2K 가변저항으로 조정, 2s는 보수적 추정 |

#### thermal_resistance 역산 과정

열 모델 미분방정식:

```
dT/dt = I² × R_winding × R_thermal - (T - T_amb) / tau_cool
```

정상상태(dT/dt = 0)에서:

```
T_ss = T_amb + I² × R_winding × R_thermal × tau_cool
```

**목표 조건**: 정격 전류(78A) 연속 운전 시 정상상태 온도가 65°C를 넘지 않아야 한다.

```
60°C = 25°C + 78² × R_winding × R_thermal × tau_cool
35 = 6084 × R_winding × R_thermal × tau_cool
```

`R_winding = 0.05Ω`, `tau_cool = 60s`로 고정하고 `R_thermal`을 역산:

```
R_thermal = 35 / (6084 × 0.05 × 60) = 35 / 18252 ≈ 0.00192
```

**0.002로 반올림** → 정격에서 T_ss ≈ 61.5°C

검증:

| 전류 | 정상상태 온도 | 65°C 초과? | 비고 |
|------|-------------|-----------|------|
| 50A (경부하) | 25 + 50² × 0.05 × 0.002 × 60 = **40.0°C** | X | 정상 |
| 78A (정격) | 25 + 78² × 0.05 × 0.002 × 60 = **61.5°C** | X | 65°C 직전 |
| 85A (약과부하) | 25 + 85² × 0.05 × 0.002 × 60 = **68.3°C** | O | 장시간 운전 시 과온도 |
| 95A (과부하) | 25 + 95² × 0.05 × 0.002 × 60 = **79.2°C** | O | Soft fuse가 먼저 트립(2s) |
| 100A (최대) | 25 + 100² × 0.05 × 0.002 × 60 = **85.0°C** | O | Soft fuse 즉시 트립 |

과부하(95A)에서 65°C 도달 시간:

```
T(t) = T_ss - (T_ss - T_0) × exp(-t/tau)
65 = 79.2 - (79.2 - 25) × exp(-t/60)
exp(-t/60) = (79.2 - 65) / 54.2 = 0.262
t = -60 × ln(0.262) ≈ 80.6s
```

→ 95A 과부하 시 **~81초 후 과온도 알람** (단, Soft fuse가 2초에 먼저 트립하므로 실제로는 과전류 폴트가 선행)

#### 가정의 한계와 리스크

1. **R_winding = 0.05Ω**: 실제 권선 저항이 이보다 높으면 발열이 더 크고, 낮으면 열 여유가 늘어남. 오차 범위 ±100% 가능
2. **tau_cool = 60s**: 방열 조건(밀폐 vs 개방, 자연 냉각 vs 강제 냉각)에 따라 크게 달라짐. 실차 밀폐 조건이면 120s 이상일 수 있음
3. **단일 노드 열 모델**: 모터 전체를 하나의 온도로 추상화. 실제로는 권선(고온) → 하우징(중온) → 외기(저온) 다단계 열 전달. 권선 피크 온도를 과소추정할 가능성
4. **역산의 순환 논리**: "정격에서 65°C 직전"이라는 목표 자체가 가정. DB130-48의 실제 열 설계는 연속 정격에서 class F 절연(155°C)까지 허용할 수 있음. 65°C는 MD2K 드라이버 기판 온도 제한이지 모터 자체 온도 제한이 아닐 가능성
5. **정상상태 도달 시간**: 5×tau = 300초(5분). 과수원 작업 시 이보다 긴 연속 운전이 예상되므로 정상상태 가정은 유효

> **결론**: 현재 열 모델은 "정성적으로 타당한 동작"을 보여주는 수준이며, 정량적 정확도는 실차 온도 데이터 확보 후 튜닝 필요.
